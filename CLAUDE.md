# Claude Slack Bridge

Two-way Slack ā†” Claude Code chat bridge, plus outbound notifications. Each Slack channel or DM maps to one persistent Claude Code session. Sessions survive bridge restarts ā€” the session ID is stored in `slack_sessions.json`.

---

## Files

| File | Purpose |
|------|---------|
| `slack_bridge.py` | Core bridge ā€” Socket Mode bot, session management, heartbeat |
| `slack_bridge_tray.pyw` | Windows system-tray launcher (green/grey dot, auto-restart) |
| `md_to_slack.py` | Markdown ā†’ Slack mrkdwn converter; also handles Block Kit tables |
| `notify.ps1` | Post outbound alerts to a Slack channel |
| `slack_sessions.json` | Runtime session store ā€” auto-created, do not edit by hand |
| `logs/slack_bridge.log` | Bridge stdout/stderr ā€” auto-created by tray launcher |

---

## How to run

**Always start via the tray launcher, not directly:**

```
pythonw slack_bridge_tray.pyw
```

This puts a green/grey dot in the Windows notification area. The tray auto-restarts the bridge on crash or clean exit. Right-click for: Restart Bridge Ā· View Log Ā· Quit.

To run without a tray (debugging):

```
python slack_bridge.py
```

## Required environment variables

```powershell
$env:SLACK_BOT_TOKEN  = "xoxb-..."   # Bot User OAuth Token
$env:SLACK_APP_TOKEN  = "xapp-..."   # App-Level Token (Socket Mode)
$env:CLAUDE_SLACK_CWD = "C:/path/to/working/dir"  # working dir for the claude CLI
```

`CLAUDE_SLACK_CWD` is the directory Claude Code runs in ā€” set it to your project root. The bridge will error on the first message if it is not set.

## Optional environment variables

```powershell
$env:CLAUDE_SLACK_CHANNEL  = "C01234567"  # restrict to one channel ID
$env:CLAUDE_SLACK_HISTORY  = "40"         # messages of channel history to inject on new sessions
$env:CLAUDE_SLACK_ADD_DIR  = "C:/other"   # extra dir passed as --add-dir to claude CLI
$env:SLACK_USER_TOKEN      = "xoxp-..."   # sets Claude's Slack status (online/offline)
$env:SLACK_NOTIFY_CHANNEL  = "#general"   # default channel for notify.ps1
$env:SLACK_ADMIN_USER      = "U01234567"  # Slack user ID allowed to run /ps-admin
```

## Slack app setup (one-time)

1. Go to https://api.slack.com/apps ā†’ **Create New App** ā†’ From scratch
2. **Socket Mode** (Features ā†’ Socket Mode) ā†’ Enable ā†’ generates the App Token (`xapp-...`)
3. **OAuth & Permissions** ā†’ Bot Token Scopes ā€” add:
   - `app_mentions:read`
   - `chat:write`
   - `chat:write.public`
   - `chat:delete` (needed for splitting long responses)
   - `im:history`
   - `im:read`
   - `im:write`
   - `channels:history`
   - `channels:read`
   - `commands`
4. **Event Subscriptions** ā†’ Subscribe to bot events:
   - `message.im`
   - `message.channels`
   - `app_mention`
5. **Slash Commands** ā€” register any commands you want (e.g. `/new`, `/stop`, `/reset`, `/restart`). With Socket Mode, no Request URL is needed.
6. **Install app to workspace** ā†’ copy the Bot Token (`xoxb-...`)

## Behaviour

**DMs:** every message triggers Claude. The entire DM = one session.

**Channels:** every message triggers Claude (bot must be a member of the channel). The entire channel = one session. On a new session, recent channel history is fetched and prepended as context.

**Thinking indicator:** while Claude works, the bot posts a placeholder with a rotating coloured bar and a whimsical verb (e.g. *Cogitating (12s)*). The verb is generated by Haiku based on the current tool. Once Claude finishes, the placeholder is updated in-place with the full response.

**Long responses:** auto-split at Slack's 3000-character limit.

**Tables:** rendered as Block Kit rich_text blocks (bold labels, italic values).

## Built-in commands

These are handled directly by the bridge ā€” no Slack slash command registration needed for `/new` and `/reset` (they can be sent as plain messages too):

| Command | Effect |
|---------|--------|
| `/new` or `/reset` | Clear saved session ā€” next message starts fresh |
| `/restart` | Bridge exits cleanly; tray relaunches it within 3 s |
| `/stop` | Terminate the in-flight Claude subprocess immediately |

Any other slash command registered in the Slack app dashboard is automatically passed through to Claude Code as message text.

## notify.ps1

Post an outbound Slack message from a script or scheduled task:

```powershell
# One-liner
.\notify.ps1 -Title "Build done" -Message "All tests passed."

# From file (markdown supported)
.\notify.ps1 -Title "Report" -MessageFile "C:\tmp\report.md" -Channel "#dev"
```

Requires `SLACK_BOT_TOKEN`. Uses `md_to_slack.py` to convert markdown to Slack mrkdwn before posting.

## Architecture notes

- **Session store:** `slack_sessions.json` maps Slack channel IDs to Claude `session_id` strings. This file is written on every message and is the only persistent state. Delete it to reset all sessions.
- **Per-channel queue:** messages for the same channel are serialised through a `queue.Queue` with a worker thread per channel. Rapid-fire messages are batched into a single Claude invocation.
- **Heartbeat:** a daemon thread updates the placeholder every 5 seconds while Claude runs. Each tool-call event triggers a background Haiku call to generate a contextual verb. There is no hard timeout ā€” use `/stop` to interrupt.
- **`md_to_slack.py`** is also a CLI tool: `echo "**bold**" | python md_to_slack.py`.
- The bridge sets `CLAUDE_VIA_SLACK=1` in the environment so Claude and its hooks can detect they are running in Slack context.

## Troubleshooting

**Bridge starts but doesn't respond:**
- Check `logs/slack_bridge.log` for errors
- Verify `SLACK_BOT_TOKEN` and `SLACK_APP_TOKEN` are set in the environment that launched the tray
- Confirm the bot is a member of the channel (invite it with `/invite @botname`)

**`CLAUDE_SLACK_CWD env var is required but not set`:**
- Set `CLAUDE_SLACK_CWD` to the directory you want Claude to run in

**Responses are blank or `_(no response)_`:**
- Check that `claude` is on the PATH and works from the terminal: `claude -p "hello" --output-format text`

**Tray icon stays grey:**
- The bridge crashed at startup ā€” open View Log from the tray menu
